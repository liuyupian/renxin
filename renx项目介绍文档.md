# 韧芯科技 - 心理韧性数字化平台项目介绍

## 一、项目基本信息

**项目名称**：韧芯科技 - 心理韧性数字化平台

**项目定位**：面向高校学生的心理韧性数字化服务平台，采用B2B2C商业模式

**核心价值**：通过数字化手段，将心理韧性培养前置化、常态化，帮助学生主动提升心理韧性，从"被动治疗"转向"主动预防"

**目标用户**：

- **C端用户**：高校学生
- **B端客户**：高校心理中心、学工部、辅导员

**技术栈**：

- **后端**：Spring Boot 3.2.5 + MyBatis-Plus 3.5.5 + MySQL 8.0 + Redis + Spring Security
- **前端**：鸿蒙ArkUI（ArkTS）原生开发
- **AI框架**：LangChain4j 0.27.1 + OpenAI/豆包大模型

------

## 二、核心功能模块

### 1. 用户管理模块

- 学生注册、登录、信息管理
- 高校管理员权限管理
- 用户类型：学生（1）、高校管理员（2）、校级管理员（3）

### 2. 心理韧性测评模块

- 基于胡月琴、甘怡群编制的**青少年心理韧性量表**（27题版）
- 支持**反向计分**（12道反向题）
- **5个维度**：目标专注、情绪控制、积极认知、家庭支持、人际协助
- 自动生成维度得分、雷达图数据、个性化建议
- 测评历史记录和趋势分析

### 3. 心理韧性工具包模块（核心AI应用）

- 5个维度 × 3个工具 = 15个专业工具
- 其中**11个工具接入AI**，支持个性化内容生成
- 工具使用记录和成长轨迹追踪

### 4. 内容推荐模块（心理健身房）

- 根据测评结果推荐心理内容
- 内容浏览记录和统计分析

### 5. 日记与成长记录模块

- 学生日记功能
- 自动生成测评成长记录
- 工具使用成长轨迹

### 6. AI心理小助手模块

- 智能对话心理助手"韧韧"
- 支持多轮对话和上下文记忆
- 高风险关键词检测和安全引导

### 7. 高校端数据看板模块

- 高校整体韧性数据概览
- 院系数据对比分析
- 关键指标趋势图
- **高风险学生预警**（数据脱敏处理）

### 8. 辅导员赋能工具箱模块

- 根据维度得分推荐干预策略
- 团体辅导方案模板管理

### 9. 训练营活动模块

- 线上线下训练营管理
- 报名管理和参与名单

### 10. 通知公告模块

- 高校公告发布和管理
- 学生公告推送

## 三、AI功能详细介绍（核心亮点）

### 3.1 AI技术架构

**AI框架选择**：LangChain4j 0.27.1

**大模型接入**：

- 支持OpenAI API
- 支持自定义API地址（如豆包等国产大模型）
- 可配置模型名称、最大Token数、温度参数

**技术实现**（AIServiceImpl.java）：

```
chatLanguageModel = OpenAiChatModel.builder()
    .baseUrl(aiConfig.getApiUrl())  // 支持自定义API地址
    .apiKey(aiConfig.getApiKey())
    .modelName(aiConfig.getModelName())
    .maxTokens(aiConfig.getMaxTokens())
    .temperature(aiConfig.getTemperature())
    .build();
```

**上下文管理**：

- 使用Redis存储对话历史
- Key格式：`chat:history:{sessionId}`
- 保留最近20条对话记录
- 24小时自动过期，平衡上下文连续性和隐私保护

### 3.2 AI心理小助手"韧韧"

**角色定位**：温暖、专业、有同理心的心理成长助手

**System Prompt设计**：

```
你是韧韧，一个温暖、专业、有同理心的心理成长助手。
 
【核心原则】
1. 用温暖、鼓励、非评判的语气与学生交流
2. 避免使用诊断性语言，不给学生贴标签
3. 专注于发展性视角，强调成长的可能性
4. 提供实用的情绪调节和自我提升建议
5. 保持边界感，不做心理咨询师的替代
 
【对话风格】
- 使用第一人称称呼（我、你）
- 适当使用表情符号增加亲和力
- 回复简洁明了，避免过长
- 多用开放式问题引导学生思考
 
【安全边界】
- 不提供心理疾病诊断
- 不提供危机干预方案
- 遇到高风险话题时温和引导寻求专业帮助
 
【回复框架】
1. 共情回应：理解学生的感受
2. 正常化：让学生知道这种感受是正常的
3. 提供视角：从积极角度看待问题
4. 给出建议：提供1-2个可操作的小建议
5. 鼓励行动：鼓励学生尝试
```

**对话流程**（第45-143行）：

1. 从Redis获取历史对话记录作为上下文
2. 构建消息列表：System Prompt + 历史对话 + 当前消息
3. 调用大模型生成回复
4. 将对话保存到Redis
5. 限制历史记录数量（20条）和过期时间（24小时）

**安全护栏机制**（第145-160行）：

- 高风险关键词检测：自杀、自残、抑郁等
- 检测到高风险关键词时，自动引导至心理援助热线
- 配置化的关键词列表，便于维护和扩展

### 3.4 AI功能的技术亮点

**1. Prompt工程专业化**

- 每个工具的Prompt都基于具体的心理学理论
- Prompt包含明确的结构要求（如提供分析、建议、话术等）
- 字数控制在100-200字，确保内容简洁实用

**2. 上下文管理**

- Redis存储对话历史，支持多轮对话
- 保留最近20条记录，避免上下文过长
- 24小时自动过期，保护用户隐私

**3. 安全护栏机制**

- 高风险关键词检测（第145-160行）
- 自动引导至心理援助热线
- 不提供心理疾病诊断和危机干预方案

**4. 成本控制**

- AI调用采用异步处理，避免阻塞主流程
- 可配置最大Token数和温度参数
- 支持缓存常见问题的AI回复（计划中）

**5. 模型可替换性**

- 支持自定义API地址，可切换OpenAI、豆包等不同大模型
- 配置化管理，无需修改代码即可更换模型

## 四、技术架构

### 4.1 后端架构

**分层架构**：

```
Controller层（控制器）→ Service层（业务逻辑）→ ServiceImpl层（实现）→ Mapper层（数据访问）→ Entity/DTO/VO（数据模型）
```

**核心组件**：

- **Spring Boot 3.2.5**：应用框架
- **MyBatis-Plus 3.5.5**：ORM框架，支持单表CRUD、分页、逻辑删除
- **MySQL 8.0**：关系型数据库
- **Redis**：缓存和会话管理
- **Spring Security**：密码加密（BCryptPasswordEncoder）
- **LangChain4j**：AI大模型集成

**开发规范**：

- 统一`Result`响应格式（code、msg、data）
- 全局异常处理
- 中文文档注释
- 中文日志打印
- 字段自动填充（create_time、update_time）
- 逻辑删除（deleted字段）
- 分页支持

### 4.2 数据库设计

**核心表结构**（15张核心表）：

1. `user`：用户表
2. `assessment_scale`：量表表
3. `assessment_question`：题目表
4. `assessment_option`：选项表
5. `assessment_record`：测评记录表
6. `assessment_answer`：答案表
7. `assessment_result`：测评结果表
8. `assessment_dimension`：维度表
9. `assessment_dimension_rule`：维度分档规则表
10. `assessment_scale_rule`：量表分档规则表
11. `assessment_question_reverse`：反向计分题目表
12. `assessment_question_dimension`：题目维度关联表
13. `tool_config`：工具配置表
14. `tool_use_log`：工具使用记录表
15. `growth_record`：成长记录表

### 4.3 前端架构

**技术选型**：鸿蒙ArkUI（ArkTS）原生开发

**技术要求**：

- 仅使用ArkTS语法和鸿蒙原生API
- 不使用WebView嵌入H5
- 不引入非鸿蒙兼容的第三方依赖
- 网络请求使用鸿蒙原生`@ohos.net.http`模块
- 在`module.json5`中声明`ohos.permission.INTERNET`网络权限

**API地址管理**：

- 所有后端接口地址统一在`ApiConfig`类中管理
- `BASE_URL`固定为后端电脑的局域网IP（如`http://10.4.55.245:8080`）
- 不使用`localhost`

## 五、商业模式

**B2B2C模式**：

- **B端**：高校订阅付费（年费5-10万元）

1. 高校订阅费（基础版5万/年，专业版8万/年，定制版10-15万/年）

**成本测算**：

- 服务器成本：约1800元/月/高校
- AI调用成本：约70元/月/高校

- 毛利率：约72%

**获客成本（CAC）**：约3万元/高校 **客户生命周期价值（LTV）**：约30万元/高校 **LTV/CAC比率**：10（健康指标）

## 七、项目总结

**韧芯科技**是一个面向高校学生的心理韧性数字化平台，采用B2B2C商业模式。项目基于胡月琴、甘怡群编制的青少年心理韧性量表，通过5个维度15个专业工具的系统化干预，帮助学生提升心理韧性。

**核心AI能力**包括：

- AI心理小助手"韧韧"：支持多轮对话，具备安全护栏机制
- 11个AI驱动的心理工具：基于认知行为疗法、非暴力沟通等心理学理论，生成个性化内容
- Prompt工程专业化：每个工具都有专门的Prompt模板，确保回复符合心理学原理

**技术亮点**包括：

- Spring Boot + MyBatis-Plus成熟企业级架构
- 鸿蒙ArkUI原生开发，符合国产化信创战略
- LangChain4j集成大模型，支持模型可替换
- Redis缓存和会话管理
- 严格的隐私保护和数据脱敏

**商业价值**包括：

- 解决高校心理健康资源不足的痛点
- B2B2C模式降低获客成本，提高用户粘性
- 清晰的盈利模式和健康的LTV/CAC比率
- 政策支持，市场需求广阔
